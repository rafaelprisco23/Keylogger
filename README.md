# Keylogger

# Projeto de Estudo: Keylogger em Ambiente Controlado

## Objetivo
Este projeto foi desenvolvido **exclusivamente em ambiente controlado** com fins acadêmicos, para compreender:
- Como funcionam keyloggers.
- Quais riscos eles representam.
- Quais técnicas podem ser usadas para detectar e mitigar esse tipo de ameaça.

## Processo Realizado
1. **Criação do Keylogger (conceitual)**
   - Desenvolvido em linguagem Python.
   - Capaz de registrar entradas do teclado em tempo real.

2. **Execução em Ambiente Controlado**
   - O programa foi rodado em uma máquina virtual isolada.
   - O keylogger foi configurado para se manter invisível ao usuário (simulação).

3. **Exportação de Dados**
   - Os dados capturados foram enviados para um servidor remoto controlado pelo pesquisador.
   - Esse processo foi feito apenas para demonstrar como atacantes poderiam exfiltrar informações.

4. **Análise**
   - Foram observados os riscos de perda de dados sensíveis.
   - Discutidas técnicas de defesa, como antivírus, monitoramento de processos e uso de firewalls.

## Conclusões
- Keyloggers representam uma ameaça significativa à privacidade.
- É fundamental que empresas e usuários adotem medidas de segurança para prevenir esse tipo de ataque.
- O estudo reforça a importância de ambientes de teste controlados para aprendizado seguro.

## Aviso
Este projeto foi realizado **apenas para fins educacionais**.  
Não deve ser utilizado em sistemas reais ou sem autorização explícita.
